# production — Stage 8 — BLOCK

## Summary

Stage 8 durability evidence is insufficient for newly introduced persistent state. The diff adds/changes durable stores and state transitions (`.vcf/project.db`, global registry cache updates, migration v2 adding `project.adopted`, `ship_release` mutating `project.state`) but provides no backup coverage, restore runbook, restore test evidence, RTO/RPO definition, or backup-failure monitoring for either the per-project DB or the global registry DB. This is backup theater risk: the release changes data that operators would need after corruption/deletion or a bad migration, yet the scoped diff contains no credible recovery path.

Stage 8 requires a documented, realistic recovery path for critical durable state; none is present in the diff.

For a store touched by schema change, Stage 8 blocks when recovery is untested or undocumented.
