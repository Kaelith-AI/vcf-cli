# security — Stage 9 — PASS

## Summary

This is a final security readiness review for the 0.4 release candidate. The diff introduces a project adoption mechanism (`vcf adopt`), per-tool LLM endpoint/model defaults in `config.yaml`, and a fix for template resolution in bundled installs. No new high-severity attack surfaces were introduced. The project adoption feature correctly uses a restricted set of allowed states and validates paths. The configuration defaults are strictly typed via Zod and cross-referenced against declared endpoints to prevent invalid routing. The template resolution fix removes a hardcoded relative path in favor of a dynamic package root discovery, reducing the risk of ENOENT errors in production without introducing directory traversal vulnerabilities. The state transition to 'shipped' in `ship_release` is correctly gated on the success of the underlying release command. All changes are consistent with the project's existing trust model.
