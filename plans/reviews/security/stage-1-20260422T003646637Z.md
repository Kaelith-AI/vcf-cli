## Summary

The changes introduce a project 'adoption' mechanism, per-tool LLM endpoint/model defaults in the configuration, and a state transition to 'shipped' upon successful release. The security scope remains consistent with the existing VCF architecture: the tool operates as a local CLI/MCP server managing local SQLite databases and interacting with external LLM providers. The trust boundaries are well-defined: the boundary between the local system (user-controlled) and the external LLM providers (semi-trusted) is maintained, and the new configuration defaults simply streamline the resolution of these boundaries without introducing new untrusted input paths. The 'adopt' feature introduces a new way to initialize project metadata but does not bypass existing security controls as it primarily writes to a local `.vcf/project.db` and a global registry.
